## [2026-06-23] Tribes2-Style Networking Integration - Phase 48

### Bug Fixes
- **Fixed event timestamp overflow**
  - Date.now() timestamps exceed 32-bit max value (4294967295)
  - Changed to relative timestamps: timestamp - (Date.now() - 60000)
  - Assumes ~1 minute window for event validity
  - Applied to PositionEvent and ShotEvent on client and server
  - Benefits: Timestamps fit in 32 bits, no overflow errors

## [2026-06-23] Tribes2-Style Networking Integration - Phase 46

### Bug Fixes
- **Fixed BitStream 32-bit integer overflow error**
  - JavaScript bit shift limitation: (1 << 32) = 0
  - Fixed by using 0xFFFFFFFF directly for 32-bit max value
  - Applied to both client and server BitStream implementations
  - Benefits: No more runtime errors when writing timestamps

- **Fixed server JSON/binary message parsing**
  - Server was trying to parse binary packets as JSON
  - Added first-byte check (0x7B = '{' for JSON)
  - Routes JSON messages to MessageHandler, binary to Tribes2Networking
  - Benefits: Clean separation of message protocols

- **Fixed client join message protocol**
  - Client was sending binary packets before JSON join
  - Now sends JSON join message first, then starts StreamManager
  - Benefits: Proper connection handshake sequence

### Event System Implementation
- **Added concrete event types to EventManager**
  - PositionEvent: Player position/rotation updates (non-guaranteed)
  - ShotEvent: Shot events with target tracking (guaranteed)
  - Implemented on both client and server
  - Factory method for event instantiation by type
  - Benefits: Type-safe event handling with proper serialization

- **Updated Tribes2Adapter to use EventManager events**
  - Removed duplicate event class definitions
  - Imports PositionEvent and ShotEvent from EventManager
  - Added callbacks for position and shot events from server
  - Benefits: Single source of truth for event types

- **Updated server Tribes2Networking to use EventManager events**
  - Removed duplicate event class definitions
  - Imports PositionEvent and ShotEvent from EventManager
  - Benefits: Consistent event types across client/server

- **Implemented StreamManager event unpacking with factory**
  - EventManager.createEvent() instantiates events by type
  - Handles PositionEvent (type 1) and ShotEvent (type 2)
  - Logs warnings for unknown event types
  - Benefits: Extensible event system

### Server Integration
- **Wired Tribes2Networking to MessageHandler**
  - Added onEvent() callback for event forwarding
  - Added onMove() callback for move forwarding
  - Server constructor wires callbacks with TODOs for format conversion
  - Benefits: Path for Tribes2 events to reach game logic

### Specification Updates
